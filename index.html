Ch2 第2章 进程管理（全部在考试范围内 ✅）

2.1 进程的定义与特征（必考概念）

进程定义：进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。

进程与程序的区别：

  • 程序是指令的集合，静态概念；进程是程序在处理机上的一次执行过程，动态概念
  • 程序长期存在；进程有生命周期（创建、活动、消亡）
  • 程序仅是指令的有序集合；进程由程序、数据和进程控制块（PCB）组成
  • 进程与程序不是一一对应的：同一程序可对应多个进程，一个进程可执行多个程序

2.2 进程的三种基本状态

        就绪 ──────→ 执行
         ↑            │
         │     时间片完│
         │            ↓
        I/O完成 ←─── 阻塞
                    I/O请求
  • 就绪：进程已分配到除CPU以外的所有必要资源，只要再获得CPU就可以执行
  • 执行：进程已获得CPU，正在执行
  • 阻塞：正在执行的进程由于发生某事件而暂时无法继续执行时放弃CPU

2.3 进程控制块 PCB（重点）

PCB 是进程存在的唯一标志，包含：进程标识符（pid）、进程状态、现场信息（寄存器等）、优先级、阻塞原因、程序地址、同步机制、资源清单、链接指针。

2.4 线程的基本概念

  • 线程是调度和分派的基本单位
  • 进程是拥有资源的基本单位
  • 一个进程可以有多个线程，它们共享进程的资源

2.5 PV 操作（计算题 · 10分）

核心要点

  • PV操作有共享变量信号量存在，所以其程序是一个临界区
  • PV操作是原子操作，执行时不可中断
  • 信号量只能通过PV操作改变其值
  • 互斥的模式：PV总是成对出现在同一进程中
  • 同步的模式：PV总是成对出现在不同进程中
  • 前驱关系：有多少前驱关系设置多少个信号量（初值0）；有多少前驱做多少P操作，有多少后继做多少V操作

经典例题：爸爸/女儿/儿子 水果盘问题（课件原题）

semaphore s, so, sa = 1, 0, 0;  // s:盘子互斥, so:桔子, sa:苹果

father() {
    while(1) {
        wait(s);           // P(s) 申请盘子
        将水果放入盘中；
        if (放入的是桔子) signal(so);    // V(so) 通知儿子
        else               signal(sa);    // V(sa) 通知女儿
    }
}

son() {
    while(1) {
        wait(so);          // P(so) 等桔子
        从盘中取出桔子；
        signal(s);         // V(s) 释放盘子
        吃桔子；
    }
}

daughter() {
    while(1) {
        wait(sa);          // P(sa) 等苹果
        从盘中取出苹果；
        signal(s);         // V(s) 释放盘子
        吃苹果；
    }
}
💡 本质
生产者-消费者问题的一种变形，生产者放入的产品有两类，消费者也有两类，每类消费者只消费其中固定的一类产品。

另一个例题：单缓冲区采集与计算

semaphore empty, full = 1, 0;

process1() {  // 采集进程
    while(1) {
        wait(empty);    // P(empty)
        critical section  // 将数据送入缓冲区
        signal(full);   // V(full)
    }
}

process2() {  // 计算进程
    while(1) {
        wait(full);     // P(full)
        critical section  // 从缓冲区取出数据计算
        signal(empty);  // V(empty)
    }
}

2. 短作业优先 SJF

从后备队列中选择一个或若干个估计运行时间最短的作业。特点：降低平均周转时间，提高吞吐量；对长作业不利（饥饿）。

3. 高响应比优先（重点）

响应比 Rp =（等待时间 + 服务时间）/ 服务时间
  • 短作业RP大（优先执行）
  • 服务时间相同的进程间相当于FCFS
  • 长作业等待一段时间后仍能得到服务（不会饥饿）

4. 优先级调度

静态优先权：整个运行期不变，简单但低优先权作业可能长期不被调度。

动态优先权：如响应比优先，优先权随执行时间下降、随等待时间升高。

3.2 死锁（概念分析题）

死锁的四个必要条件

  1. 互斥条件：资源的临界性（资源的固有属性，不能避免）
  2. 请求和保持条件：进程已占有资源，又申请新资源
  3. 不剥夺条件：进程已获得的资源不能被剥夺
  4. 环路等待条件：存在进程-资源的循环等待链

死锁的处理方法

1. 预防死锁：破坏四个必要条件之一

  • 摒弃请求和保持：资源一次性分配（全分配，全释放）→ AND信号量
  • 摒弃不剥夺：新申请不能满足则释放已获得资源
  • 摒弃环路：资源的有序分配法（为资源编号）

2. 避免死锁：银行家算法

  • 安全状态：能找到安全序列的状态
  • 不安全状态：找不到安全序列的状态

3. 检测与解除

资源分配图

用有向图描述进程和资源之间的分配和请求关系。圆圈表示进程，方框表示资源，方框中的圆点表示资源实例。

💡 关键结论
如果图中存在环路，且环路中的每类资源只有一个实例，则发生死锁。

1 死锁的四个必要条件
题目

什么是死锁？产生死锁的四个必要条件是什么？哪些条件可以破坏？如何破坏？

答案

死锁：多个进程因竞争资源而无限期等待，无法继续执行的状态。

四个必要条件：

  1. 互斥条件：资源的临界性（固有属性，不能破坏）

  3. 不剥夺条件：进程已获得的资源不能被剥夺 → 破坏：新申请不能满足则释放已获得资源

2 银行家算法
题目

什么是安全状态？什么是不安全状态？银行家算法的基本思想是什么？

答案

安全状态：能找到安全序列的状态，即按某种顺序并发进程都能达到获得最大资源而顺序完成。

不安全状态：找不到安全序列的状态。

银行家算法：一种能够避免死锁的调度算法。当进程申请资源时，先假设分配，然后检查系统是否仍处于安全状态。如果是，则真正分配；否则，拒绝分配，让进程等待。

答案

不一定。资源分配图中存在环路是死锁的必要条件，但不是充分条件。

题目

解答

3-12 资源分配图化简
题目

如何用资源分配图化简法判断系统是否死锁？

解答

资源分配图化简法：

  1. 找一个既不孤立又不阻塞的进程（有请求边且请求的资源可满足）
  2. 假设该进程获得所需资源并执行完毕，消去其请求边和分配边，使之成为孤立点
  3. 重复上述步骤，直到所有进程都成为孤立点
  4. 若不能化简所有边，则系统处于死锁状态

4.1 程序的装入和链接

1. 绝对装入

编译后，装入前已产生了绝对地址（内存地址），装入时不再作地址重定位。只适合单道程序环境。

2. 可重定位装入（静态重定位）

地址转换在装入时一次完成，由软件实现（重定位装入程序完成）。缺点：不允许程序运行时在内存中移动位置。

3. 动态运行时装入（动态重定位）⭐重点

相对地址向绝对地址的转换在运行时完成。需要硬件地址变换机构：重定位寄存器的支持。

  • 优点：进程可以移动，支持紧凑
  • 物理地址 = 逻辑地址 + 重定位寄存器值
💡 关键结论
紧凑必须配合动态重定位才能实现。因为紧凑把进程搬家后，只有动态重定位能在运行时把逻辑地址正确转换成新的物理地址。

4.2 连续分配方式

1. 单一连续分配

内存分为系统区 + 用户区。应用程序装入用户区，可使用用户区全部空间。最简单，适用于单用户、单任务的OS。缺点：对要求内存空间少的程序，造成内存浪费。

2. 固定分区分配

将内存划分成若干个连续区域（分区），每个分区只能存储一个程序。分区大小相等或不相等（不相等利用率更高）。优点：简单；缺点：有内碎片（分区内剩余空间）。

3. 可变分区分配（动态分区）⭐重点

内存不预先划分好，当作业装入时，根据作业需求动态分配。数据结构：空闲分区表、空闲分区链。

算法策略特点
首次适应从低地址开始，找第一个够大的低地址使用频繁，碎片多
循环首次适应从上次找到的位置继续找分配均匀，缺乏大空闲分区
最佳适应找最小的满足需求的分区减少浪费，但产生很多小碎片
最坏适应找最大的分区减少小碎片，但大作业难满足

4. 可重定位分区分配（紧凑）⭐重点

问题：连续分配产生大量外部碎片（分散的小空闲区），总量够但连续空间不够。

紧凑：将内存中的进程移动，使它们相邻，空出大块连续空间。

条件：必须采用动态重定位，因为进程搬家后物理地址变了。

4.3 基本分页存储管理方式

逻辑地址 → 物理地址

基本分页：

  • 逻辑地址 A，页面大小 L
  • 页号 P = INT[A / L]
  • 页内偏移 d = A MOD L
  • 物理地址 = 页表[P] × L + d
💡 位运算技巧
当页面尺寸为2的幂时，可以直接用位运算拆分页号和页内地址，无需除法。
例：页大小1KB = 210，逻辑地址4101 = 二进制 000100 0000000101
→ 页号 = 000100 = 4，页内地址 = 0000000101 = 5

快表 TLB

高速缓存，存放最近访问的页表项。若命中则无需访问内存中的页表，加快地址变换速度。

两级页表

将页表再分页，解决页表过长的问题。外层页表 → 内层页表 → 物理块。

4.4 基本分段存储管理方式

段是逻辑单位（按程序的逻辑结构划分），页是物理单位。段长不固定，由用户指定。

段表：段号 | 段长 | 基址 | 存取控制

地址变换：段号查段表 → 段内地址与段长比较（越界检查）→ 物理地址 = 基址 + 段内地址

  6. 恢复CPU现场，重新执行被中断的指令

  • 多次性：进程可分批装入内存

  • 页面太大：内碎片增加；减少页表大小但降低内存利用率

题目

8.1 FAT 文件分配表

FAT12

  • 每个FAT表项12位，最多212个表项
  • 以盘块为基本分配单位时，最大分区容量 = 212 × 512B = 2MB
  • 整个磁盘最大容量 = 4 × 2MB = 8MB

FAT16

  • 每个FAT表项16位，最多65536个簇
  • 可管理最大分区空间 = 216 × 64 × 512 = 2GB
  • 簇内碎片大，浪费10%~20%空间

FAT32

  • 每个FAT表项占4字节（实际用28位）
  • 簇大小4KB时，单个最大磁盘空间 = 4KB × 228 = 1TB
  • 可用较小簇，减少簇内碎片
📌 总结
FAT12 → FAT16 → FAT32：表项位数增大，可管理磁盘容量增大。但FAT表存储开销也随之增大。

8.2 混合索引（UNIX inode）

UNIX System V 的inode中设13个地址项：

  • 直接地址：i.addr(0) ~ i.addr(9)，共10个，直接指向数据块
  • 一次间接：i.addr(10)，指向一个索引块，该索引块存放数据块地址
  • 二次间接：i.addr(11)，指向一个索引块，该索引块指向多个索引块
  • 三次间接：i.addr(12)，三级索引

最大文件大小计算

设盘块大小为4KB，每个地址占4字节：

  • 一个索引块可存 4KB/4B = 1024 个地址
  • 直接：10 × 4KB = 40KB
  • 一次间接：1024 × 4KB = 4MB
  • 二次间接：1024 × 1024 × 4KB = 4GB
  • 三次间接：10243 × 4KB = 4TB

8-1 FAT12容量计算
题目

FAT12文件系统中，FAT表项占12位，以盘块（512B）为基本分配单位。求最大分区容量和最大磁盘容量。

解答

FAT12最多有 2^12 = 4096 个表项

最大分区容量 = 4096 × 512B = 2MB

整个磁盘最多4个分区 → 最大磁盘容量 = 4 × 2MB = 8MB

8-2 FAT16容量计算
题目

FAT16文件系统中，FAT表项占16位，每簇64个盘块（每个盘块512B）。求最大分区空间，并分析簇内碎片问题。

解答

FAT16最多有 2^16 = 65536 个簇

每簇 = 64 × 512B = 32KB

最大分区空间 = 65536 × 32KB = 2GB

簇内碎片问题：文件大小不是簇大小的整数倍时，最后一个簇会产生内部碎片。平均浪费约半个簇，即16KB。对于大量小文件，浪费可达10%~20%。

8-3 FAT32容量计算
题目

FAT32文件系统中，FAT表项占4字节（实际使用28位），簇大小为4KB。求单个最大磁盘空间。

解答

FAT32最多有 2^28 = 268435456 个簇

最大磁盘空间 = 268435456 × 4KB = 1TB

FAT32可用较小的簇（如4KB），相比FAT16的32KB簇，大幅减少了簇内碎片。

Ch2 第2章 进程管理

2.1 进程的定义与特征（必考概念）

进程定义：进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。

进程与程序的区别：

  • 程序是指令的集合，静态概念；进程是程序在处理机上的一次执行过程，动态概念
  • 程序长期存在；进程有生命周期（创建、活动、消亡）
  • 程序仅是指令的有序集合；进程由程序、数据和进程控制块（PCB）组成
  • 进程与程序不是一一对应的：同一程序可对应多个进程，一个进程可执行多个程序

2.2 进程的三种基本状态

        就绪 ──────→ 执行
         ↑            │
         │     时间片完│
         │            ↓
        I/O完成 ←─── 阻塞
                    I/O请求
  • 就绪：进程已分配到除CPU以外的所有必要资源，只要再获得CPU就可以执行
  • 执行：进程已获得CPU，正在执行
  • 阻塞：正在执行的进程由于发生某事件而暂时无法继续执行时放弃CPU

2.3 进程控制块 PCB（重点）

PCB 是进程存在的唯一标志，包含：进程标识符（pid）、进程状态、现场信息（寄存器等）、优先级、阻塞原因、程序地址、同步机制、资源清单、链接指针。

2.4 线程的基本概念

  • 线程是调度和分派的基本单位
  • 进程是拥有资源的基本单位
  • 一个进程可以有多个线程，它们共享进程的资源

2.5 PV 操作（计算题 · 10分）

核心要点

  • PV操作有共享变量信号量存在，所以其程序是一个临界区
  • PV操作是原子操作，执行时不可中断
  • 信号量只能通过PV操作改变其值
  • 互斥的模式：PV总是成对出现在同一进程中
  • 同步的模式：PV总是成对出现在不同进程中
  • 前驱关系：有多少前驱关系设置多少个信号量（初值0）；有多少前驱做多少P操作，有多少后继做多少V操作

经典例题：爸爸/女儿/儿子 水果盘问题（课件原题）

桌上有一空盘，允许存放一只水果，爸爸可向盘内放苹果或桔子，儿子专等吃桔子，女儿专等吃苹果。

semaphore s, so, sa = 1, 0, 0;  // s:盘子互斥, so:桔子, sa:苹果

father() {
    while(1) {
        wait(s);           // P(s) 申请盘子
        将水果放入盘中；
        if (放入的是桔子) signal(so);    // V(so) 通知儿子
        else               signal(sa);    // V(sa) 通知女儿
    }
}

son() {
    while(1) {
        wait(so);          // P(so) 等桔子
        从盘中取出桔子；
        signal(s);         // V(s) 释放盘子
        吃桔子；
    }
}

daughter() {
    while(1) {
        wait(sa);          // P(sa) 等苹果
        从盘中取出苹果；
        signal(s);         // V(s) 释放盘子
        吃苹果；
    }
}
💡 本质
生产者-消费者问题的一种变形，生产者放入的产品有两类，消费者也有两类，每类消费者只消费其中固定的一类产品。

另一个例题：单缓冲区采集与计算

semaphore empty, full = 1, 0;

process1() {  // 采集进程
    while(1) {
        wait(empty);    // P(empty)
        critical section  // 将数据送入缓冲区
        signal(full);   // V(full)
    }
}

process2() {  // 计算进程
    while(1) {
        wait(full);     // P(full)
        critical section  // 从缓冲区取出数据计算
        signal(empty);  // V(empty)
    }
}

Ch3 第3章 处理机调度与死锁

3.1 调度算法（计算题 · 10分）

🚫 排除不考
轮转法（RR）、实时调度（EDF、松弛度）、多级反馈队列

1. 先来先服务 FCFS

按作业/进程到达的先后顺序进行调度。

特点：简单，有利于长作业，对短作业不利。

计算公式：周转时间 = 完成时间 - 到达时间；带权周转时间 = 周转时间 / 服务时间

结果

平均周转时间 = (4+6+10+11+14)/5 = 9

平均带权周转时间 = (1+2+2+5.5+3.5)/5 = 2.8

2. 短作业优先 SJF

从后备队列中选择一个或若干个估计运行时间最短的作业，将它们调入内存运行。

特点：降低平均周转时间，提高吞吐量；对长作业不利（饥饿）；估计时间不易确定。

同上例的SJF结果（假设同时到达）：平均周转时间 = 8，平均带权周转时间 = 2.1

3. 高响应比优先（重点）

响应比 Rp =（等待时间 + 服务时间）/ 服务时间

特点：

  • 短作业RP大（优先执行）
  • 服务时间相同的进程间相当于FCFS
  • 长作业等待一段时间后仍能得到服务（不会饥饿）

4. 优先级调度

静态优先权：整个运行期不变，简单但低优先权作业可能长期不被调度。

动态优先权：如响应比优先，优先权随执行时间下降、随等待时间升高。

3.2 死锁（概念分析题）

死锁的四个必要条件

  1. 互斥条件：资源的临界性（资源的固有属性，不能避免）
  2. 请求和保持条件：进程已占有资源，又申请新资源
  3. 不剥夺条件：进程已获得的资源不能被剥夺
  4. 环路等待条件：存在进程-资源的循环等待链

死锁的处理方法

1. 预防死锁：破坏四个必要条件之一

  • 摒弃请求和保持：资源一次性分配（全分配，全释放）→ AND信号量
  • 摒弃不剥夺：新申请不能满足则释放已获得资源
  • 摒弃环路：资源的有序分配法（为资源编号）

2. 避免死锁：银行家算法

  • 安全状态：能找到安全序列的状态
  • 不安全状态：找不到安全序列的状态

3. 检测与解除

资源分配图

用有向图描述进程和资源之间的分配和请求关系。圆圈表示进程，方框表示资源，方框中的圆点表示资源实例。

💡 关键结论
如果图中存在环路，且环路中的每类资源只有一个实例，则发生死锁。

Ch4 第4章 存储器管理

4.1 地址变换（计算题 · 10分）

逻辑地址 → 物理地址

基本分页：

  • 逻辑地址 A，页面大小 L
  • 页号 P = INT[A / L]
  • 页内偏移 d = A MOD L
  • 物理地址 = 页表[P] × L + d
💡 位运算技巧
当页面尺寸为2的幂时，可以直接用位运算拆分页号和页内地址，无需除法。
例：页大小1KB = 210，逻辑地址4101 = 二进制 000100 0000000101
→ 页号 = 000100 = 4，页内地址 = 0000000101 = 5

快表 TLB

高速缓存，存放最近访问的页表项。若命中则无需访问内存中的页表，加快地址变换速度。

两级页表

将页表再分页，解决页表过长的问题。查找路径：外层页表 → 内层页表 → 物理块。

缓冲管理（单缓冲、双缓冲、循环缓冲、缓冲池）

6.1 I/O 控制基本方式（4种）—— 重点概念

方式CPU干预程度传输单位适用场景
程序I/O（轮询/查询）完全干预字/字节简单低速
中断驱动每字中断字/字节中低速
DMA（直接存储器访问）开始/结束数据块高速
I/O通道发命令即可一组块大量数据

1. 程序I/O方式（轮询/查询方式）

CPU不断查询设备状态，直到设备就绪。CPU完全干预I/O，效率极低。适用于简单、低速设备。

2. 中断驱动方式

设备就绪后向CPU发中断信号，CPU响应后处理。CPU无需轮询，可以做其他事。每传一个数据就要中断一次，适合中低速设备。

3. DMA方式（直接存储器访问）

在DMA控制器的控制下，数据直接在设备和内存之间传输。无需CPU干预，只在开始和结束时需要CPU。适合高速设备（磁盘等）。以数据块为单位传输。

4. I/O通道方式

使用专门的I/O处理器（通道）执行通道程序。通道可以独立执行I/O操作，CPU只需发命令。进一步减轻CPU负担，适合大量数据传输。

6.3 SPOOLing 技术

假脱机技术：将独占设备改造成共享设备。利用磁盘作为缓冲，多个用户可以同时请求使用打印机（实际是排队）。

6.4 设备独立性

用户程序使用逻辑设备名，与物理设备无关。操作系统负责逻辑设备到物理设备的映射。

Ch7 第7章 文件管理

7.1 文件目录

  • 单级目录：简单但文件名不能重复
  • 两级目录：主文件目录 + 用户文件目录
  • 多级目录（树形目录）：现代OS普遍采用

7.2 文件逻辑结构

1. 顺序文件

  • 串结构：按存入时间先后排序
  • 顺序结构：按用户指定的关键字排序
  • 优点：批量存取效率高，只有顺序文件能存储在磁带上
  • 缺点：插入删除困难，交互应用性能差

2. 索引文件

  • 为变长记录文件建立索引表
  • 索引表按关键字排序，是定长记录的顺序文件
  • 优点：查找速度快，插入删除方便
  • 缺点：增加存储开销

7.3 索引节点（inode）

在UNIX中，文件目录项只包含文件名和指向inode的指针。

inode 存放文件的元数据：文件类型、大小、权限、时间戳、盘块地址等。

文件共享可以通过共享inode实现（硬链接）。

Ch8 第8章 磁盘存储器的管理

⚠️ 注意范围
考试范围到 8.1 FAT文件分配表 为止，8.1之后的内容不考。

8.1 FAT 文件分配表

FAT12

  • 每个FAT表项12位，最多212个表项
  • 以盘块为基本分配单位时，最大分区容量 = 212 × 512B = 2MB
  • 整个磁盘最大容量 = 4 × 2MB = 8MB

FAT16

  • 每个FAT表项16位，最多65536个簇
  • 可管理最大分区空间 = 216 × 64 × 512 = 2GB
  • 簇内碎片大，浪费10%~20%空间

FAT32

  • 每个FAT表项占4字节（实际用28位）
  • 簇大小4KB时，单个最大磁盘空间 = 4KB × 228 = 1TB
  • 可用较小簇，减少簇内碎片
📌 总结
FAT12 → FAT16 → FAT32：表项位数增大，可管理磁盘容量增大。但FAT表存储开销也随之增大。

8.2 混合索引（UNIX inode）

UNIX System V 的inode中设13个地址项：

  • 直接地址：i.addr(0) ~ i.addr(9)，共10个，直接指向数据块
  • 一次间接：i.addr(10)，指向一个索引块，该索引块存放数据块地址
  • 二次间接：i.addr(11)，指向一个索引块，该索引块指向多个索引块
  • 三次间接：i.addr(12)，三级索引

最大文件大小计算

设盘块大小为4KB，每个地址占4字节：

  • 一个索引块可存 4KB/4B = 1024 个地址
  • 直接：10 × 4KB = 40KB
  • 一次间接：1024 × 4KB = 4MB
  • 二次间接：1024 × 1024 × 4KB = 4GB
  • 三次间接：10243 × 4KB = 4TB

8.3 文件物理结构比较

分配方式优点缺点
连续分配速度快，支持随机访问产生碎片，不利于增长
链接分配无碎片，利于增长随机访问慢，指针占用空间

有效访问时间 = (1−p) × t + p × fp=缺页率, t=内存访问时间, f=缺页中断时间

## 第2章 进程管理（必考概念 + 计算题）

### 2.1 进程的定义与特征（必考概念）

**进程定义**：进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。

**进程与程序的区别**：
1. 程序是指令的集合，**静态概念**；进程是程序在处理机上的一次执行过程，**动态概念**
2. 程序长期存在；进程有生命周期（创建、活动、消亡）
3. 程序仅是指令的有序集合；进程由**程序、数据和进程控制块（PCB）**组成
4. 进程与程序**不是一一对应**的：同一程序可对应多个进程，一个进程可执行多个程序

### 2.2 进程的三种基本状态

```
        就绪 ──────→ 执行
         ↑            │
         │     时间片完│
         │            ↓
        I/O完成 ←─── 阻塞
                    I/O请求
```

- **就绪**：进程已分配到除CPU以外的所有必要资源，只要再获得CPU就可以执行
- **执行**：进程已获得CPU，正在执行
- **阻塞**：正在执行的进程由于发生某事件而暂时无法继续执行时放弃CPU

### 2.3 进程控制块 PCB（重点）

PCB 是进程存在的**唯一标志**，包含：进程标识符（pid）、进程状态、现场信息（寄存器等）、优先级、阻塞原因、程序地址、同步机制、资源清单、链接指针。

### 2.4 线程的基本概念

- 线程是**调度和分派的基本单位**
- 进程是**拥有资源的基本单位**
- 一个进程可以有多个线程，它们共享进程的资源

### PV 操作（计算题 · 10分）

#### 核心要点

1. PV操作有共享变量信号量存在，所以其程序是一个临界区
2. PV操作是**原子操作**，执行时不可中断
3. 信号量只能通过PV操作改变其值
4. **互斥的模式**：PV总是成对出现在**同一进程**中
5. **同步的模式**：PV总是成对出现在**不同进程**中
6. **前驱关系**：有多少前驱关系设置多少个信号量（初值0）；有多少前驱做多少P操作，有多少后继做多少V操作

#### 经典例题：爸爸/女儿/儿子 水果盘问题（课件原题）

```c
semaphore s, so, sa = 1, 0, 0;  // s:盘子互斥, so:桔子, sa:苹果

father() {
    while(1) {
        wait(s);           // P(s) 申请盘子
        将水果放入盘中；
        if (放入的是桔子) signal(so);    // V(so) 通知儿子
        else               signal(sa);    // V(sa) 通知女儿
    }
}

son() {
    while(1) {
        wait(so);          // P(so) 等桔子
        从盘中取出桔子；
        signal(s);         // V(s) 释放盘子
        吃桔子；
    }
}

daughter() {
    while(1) {
        wait(sa);          // P(sa) 等苹果
        从盘中取出苹果；
        signal(s);         // V(s) 释放盘子
        吃苹果；
    }
}
```

**本质**：生产者-消费者问题的一种变形，生产者放入的产品有两类，消费者也有两类，每类消费者只消费其中固定的一类产品。

#### 另一个例题：单缓冲区采集与计算

```c
semaphore empty, full = 1, 0;

process1() {  // 采集进程
    while(1) {
        wait(empty);    // P(empty)
        critical section  // 将数据送入缓冲区
        signal(full);   // V(full)
    }
}

process2() {  // 计算进程
    while(1) {
        wait(full);     // P(full)
        critical section  // 从缓冲区取出数据计算
        signal(empty);  // V(empty)
    }
}
```

## 第3章 处理机调度与死锁

### 🚫 排除不考

轮转法（RR）、实时调度（EDF、松弛度）、多级反馈队列

### 3.1 调度算法（计算题 · 10分）

#### 1. 先来先服务 FCFS

- 按作业/进程到达的先后顺序进行调度
- **特点**：简单，有利于长作业，对短作业不利
- 计算：周转时间 = 完成时间 - 到达时间；带权周转时间 = 周转时间 / 服务时间

**课件例题**：

| 进程 | 到达时间 | 服务时间 | FCFS完成 | 周转时间 | 带权周转 |
|------|----------|----------|----------|----------|----------|
| A    | 0        | 4        | 4        | 4        | 1        |
| B    | 1        | 3        | 7        | 6        | 2        |
| C    | 2        | 5        | 12       | 10       | 2        |
| D    | 3        | 2        | 14       | 11       | 5.5      |
| E    | 4        | 4        | 18       | 14       | 3.5      |

平均周转时间 = (4+6+10+11+14)/5 = **9**  
平均带权周转时间 = (1+2+2+5.5+3.5)/5 = **2.8**

#### 2. 短作业优先 SJF

- 从后备队列中选择一个或若干个估计运行时间最短的作业
- **特点**：降低平均周转时间，提高吞吐量；对长作业不利（饥饿）；估计时间不易确定
- 同上例的SJF结果（假设同时到达）：平均周转时间 = **8**，平均带权周转时间 = **2.1**

#### 3. 高响应比优先（重点）

**响应比 Rp =（等待时间 + 服务时间）/ 服务时间**

特点：
- 短作业RP大（优先执行）
- 服务时间相同的进程间相当于FCFS
- 长作业等待一段时间后仍能得到服务（不会饥饿）

#### 4. 优先级调度

- **静态优先权**：整个运行期不变，简单但低优先权作业可能长期不被调度
- **动态优先权**：如响应比优先，优先权随执行时间下降、随等待时间升高

### 3.2 死锁（概念分析题）

#### 死锁的四个必要条件

1. **互斥条件**：资源的临界性（资源的固有属性，不能避免）
2. **请求和保持条件**：进程已占有资源，又申请新资源
3. **不剥夺条件**：进程已获得的资源不能被剥夺
4. **环路等待条件**：存在进程-资源的循环等待链

#### 死锁的处理方法

1. **预防死锁**：破坏四个必要条件之一
   - 摒弃请求和保持：资源一次性分配（全分配，全释放）→ AND信号量
   - 摒弃不剥夺：新申请不能满足则释放已获得资源
   - 摒弃环路：资源的有序分配法（为资源编号）

2. **避免死锁**：银行家算法
   - 安全状态：能找到安全序列的状态
   - 不安全状态：找不到安全序列的状态

3. **检测与解除**

#### 资源分配图

- 用有向图描述进程和资源之间的分配和请求关系
- 圆圈表示进程，方框表示资源，方框中的圆点表示资源实例
- **如果图中存在环路，且环路中的每类资源只有一个实例，则发生死锁**

## 第4章 存储器管理

### 4.1 地址变换（计算题 · 10分）

#### 逻辑地址 → 物理地址

**基本分页**：
- 逻辑地址 A，页面大小 L
- 页号 P = INT[A / L]
- 页内偏移 d = A MOD L
- 物理地址 = 页表[P] × L + d

**当页面尺寸为2的幂时**（考试通常如此）：
- 可以直接用位运算拆分，无需除法
- 例：页大小1KB = 2^10，逻辑地址4101
  - 4101 = 2^12 + 2^2 + 2^0 = 二进制 000100 0000000101
  - 页号 = 000100 = 4，页内地址 = 0000000101 = 5
  - 即 A=4101 → (4, 5)

#### 地址变换过程

1. CPU给出逻辑地址
2. 拆分页号P和页内偏移d
3. 查页表（或快表TLB）得到物理块号
4. 物理地址 = 物理块号 × 页大小 + 页内偏移

#### 快表 TLB

- 高速缓存，存放最近访问的页表项
- 若命中则无需访问内存中的页表，加快地址变换速度

#### 两级页表

- 将页表再分页，解决页表过长的问题
- 外层页表 → 内层页表 → 物理块

#### 习题3：TLB与内存访问时间

**题目**：有一页式系统，其页表存放在主存中：

(1) 如果对主存的一次存取需要1.5μs，试问实现一次页面访问的存取时间是多少？
(2) 如果系统加有快表，平均命中率为85%，当页表项在快表中时，其查找时间可忽略为0，试问此时的存取时间是多少？

**解答**：

若页表存放在主存中，则要实现一次页面访问需两次访问主存：一次是访问页表，确定所存取页面的物理地址（称为定位）。第二次才根据该地址存取页面数据。

1. 页表在主存的存取访问时间 = 1.5 × 2 = **3 μs**
2. 增加快表后的存取访问时间 = 0.85 × 1.5 + (1-0.85) × 2 × 1.5 = **1.725 μs**

## 第5章 虚拟存储器

### 🚫 排除不考

Clock置换算法（简单Clock和改进型Clock）

### 5.1 页面置换算法（计算题 · 10分）

**考试范围**：FIFO、OPT、LRU、缺页率

#### 1. 最佳置换算法 OPT

- 选择**以后永不使用**或**在最长时间内不再被访问**的页面淘汰
- 理论最优，但无法实现（需要预知未来），用于比较评价

#### 2. 先进先出 FIFO

- 选择**最早进入内存**的页面淘汰
- 可能出现 **Belady异常**：物理块增加，缺页率反而增加

#### 3. 最近最久未使用 LRU

- 选择**最近一段时间最久未被访问**的页面淘汰
- 实现方式：移位寄存器或栈
- 性能接近OPT，是实际常用的算法

#### 缺页率计算

- **缺页率** = 缺页次数 / 总访问次数
- 缺页中断处理过程：
  1. 保留CPU现场
  2. 从外存找到缺页
  3. 内存满则选择一页换出
  4. 若被换出页被修改过，写回外存
  5. 从外存读入缺页
  6. 修改页表和快表

#### 请求分页的基本概念

- **局部性原理**：时间局部性（循环）+ 空间局部性（顺序）
- 虚拟存储器的特征：离散性、多次性、对换性、虚拟性
- **页表项**：页号 | 物理块号 | 状态位P | 访问字段A | 修改位M | 外存地址

#### 有效访问时间

```
有效访问时间 = (1-p) × t + p × f

其中：p为缺页率，t为内存访问时间，f为缺页中断时间
```

#### 习题2：二维数组缺页分析

**题目**：有一个二维数组：
```pascal
Var A: ARRAY[1..100, 1..100] OF integer;
```
按行优先顺序存储。对于一采用LRU置换算法的页式虚拟存储器系统，假设每页可存放200个整数。若分配给一进程的内存块数为3，其中一块用来装入程序和变量i、j，另外两块专门用来存放数组（不作它用），且程序段已在内存，但数据页尚未装入内存。请分别就下列程序计算执行过程中的缺页次数：

```pascal
程序1：                          程序2：
FOR i:=1 TO 100 DO             FOR j:=1 TO 100 DO
  FOR j:=1 TO 100 DO             FOR i:=1 TO 100 DO
    A[i,j]:=0;                    A[i,j]:=0;
```

**解答**：

对程序1，首次缺页中断（访问A[1,1]时产生）将装入数组的第1、2行共200个整数，由于程序是按行对数组进行访问，只有在处理完200个整数后才会再次产生缺页中断，以后每调入一页，也能处理200个整数，因此处理100×100个整数共将发生 **50次缺页**。

对程序2，首次缺页中断同样将装入数组的第1、2行共200个整数，但由于程序是按列对数组进行访问，因此在处理完2个整数后又会再次产生缺页中断，以后每调入一页，也只能处理2个整数，因此，处理100×100个整数共将产生 **5000次缺页**。

> 💡 **结论**：存储顺序与访问顺序一致时（程序1按行存储按行访问），缺页率大大降低。这体现了**空间局部性原理**。

#### 习题3：缺页率与有效访问时间

**题目**：现有一请求调页系统，页表保存在寄存器中。若有一个被替换的页未被修改，则处理一个缺页中断需要8ms；若被替换页修改过，则处理一个缺页中断需要20ms。内存存取时间为1μs，访问页表时间可忽略不计。假定70%被替换的页被修改过，为保证有效存取时间不超过2μs，可接受的最大缺页率是多少？

**解答**：

设p表示缺页率，则有效时间不超过2μs可表示为：

```
(1-p) × 1μs + p × (0.7 × 20ms + 0.3 × 8ms + 1μs) ≤ 2μs

(1-p) × 1 + p × (0.7 × 20000 + 0.3 × 8000 + 1) ≤ 2
1 + 16399.1p ≤ 2
p ≤ 1/16399.1 ≈ 0.00006
```

即可接受的最大缺页率为 **0.00006**（约0.006%）。

#### 习题5：段表访问分析

**题目**：假如一个程序的段表如下，其中存在位为1表示段在内存，对于下面指令，在执行时会产生什么样的结果？

| 段号 | 存在位 | 内存始址 | 段长 | 存取控制 |
|------|--------|---------|------|---------|
| 0    | 0      | 500     | 100  | W       |
| 1    | 1      | 1000    | 30   | R       |
| 2    | 1      | 3000    | 200  | E       |
| 3    | 1      | 8000    | 80   | R       |
| 4    | 0      | 5000    | 40   | R       |

指令：
1. STORE R1, [0, 70]
2. STORE R1, [1, 20]
3. LOAD R1, [3, 20]
4. LOAD R1, [3, 100]
5. JMP [2, 100]

**解答**：

1. **STORE R1, [0, 70]**：段0存在位为0，**缺段中断**
2. **STORE R1, [1, 20]**：段1存在位为1，但存取控制为R（只读），**存取控制保护中断**
3. **LOAD R1, [3, 20]**：段3存在位为1，段内地址20 < 80，**正常执行**
4. **LOAD R1, [3, 100]**：段3段长80，100 > 80，**越界中断**
5. **JMP [2, 100]**：段2存取控制为E（执行），段内地址100 < 200，**正常执行**

## 第6章 输入输出系统

### 🚫 排除不考

缓冲管理（单缓冲、双缓冲、循环缓冲、缓冲池）

### 6.1 I/O 控制基本方式（4种）—— 重点概念

#### 1. 程序I/O方式（轮询/查询方式）

- CPU不断查询设备状态，直到设备就绪
- **CPU完全干预I/O**，效率极低
- 适用于：简单、低速设备

#### 2. 中断驱动方式

- 设备就绪后向CPU发中断信号，CPU响应后处理
- CPU无需轮询，可以做其他事
- 每传一个数据就要中断一次，适合**中低速**设备

#### 3. DMA方式（直接存储器访问）

- 在**DMA控制器**的控制下，数据直接在设备和内存之间传输
- **无需CPU干预**，只在开始和结束时需要CPU
- 适合**高速**设备（磁盘等）
- 以**数据块**为单位传输

#### 4. I/O通道方式

- 使用专门的**I/O处理器（通道）**执行通道程序
- 通道可以独立执行I/O操作，CPU只需发命令
- 进一步减轻CPU负担，适合**大量数据**传输

**总结比较**：

| 方式 | CPU干预程度 | 传输单位 | 适用场景 |
|------|------------|----------|----------|
| 程序I/O | 完全干预 | 字/字节 | 简单低速 |
| 中断 | 每字中断 | 字/字节 | 中低速 |
| DMA | 开始/结束 | 数据块 | 高速 |
| 通道 | 发命令即可 | 一组块 | 大量数据 |

### 6.3 SPOOLing 技术

- **假脱机技术**：将独占设备改造成共享设备
- 利用磁盘作为缓冲，多个用户可以同时请求使用打印机（实际是排队）

### 6.4 设备独立性

- 用户程序使用**逻辑设备名**，与物理设备无关
- 操作系统负责逻辑设备到物理设备的映射

## 第7章 文件管理

### 7.1 文件目录

- **单级目录**：简单但文件名不能重复
- **两级目录**：主文件目录 + 用户文件目录
- **多级目录（树形目录）**：现代OS普遍采用

### 7.2 文件逻辑结构

#### 1. 顺序文件

- **串结构**：按存入时间先后排序
- **顺序结构**：按用户指定的关键字排序
- **优点**：批量存取效率高，只有顺序文件能存储在磁带上
- **缺点**：插入删除困难，交互应用性能差

#### 2. 索引文件

- 为变长记录文件建立索引表
- 索引表按关键字排序，是定长记录的顺序文件
- **优点**：查找速度快，插入删除方便
- **缺点**：增加存储开销

### 7.3 索引节点（inode）

- 在UNIX中，文件目录项只包含文件名和指向inode的指针
- **inode** 存放文件的元数据：文件类型、大小、权限、时间戳、盘块地址等
- 文件共享可以通过共享inode实现（硬链接）

## 第8章 磁盘存储器的管理

### ⚠️ 注意范围

考试范围到 **8.1 FAT文件分配表** 为止，8.1之后的内容不考。

### 8.1 FAT 文件分配表

#### FAT12
- 每个FAT表项12位，最多2^12个表项
- 以盘块为基本分配单位时，最大分区容量 = 2^12 × 512B = **2MB**
- 整个磁盘最大容量 = 4 × 2MB = **8MB**

#### FAT16
- 每个FAT表项16位，最多65536个簇
- 可管理最大分区空间 = 2^16 × 64 × 512 = **2GB**
- 簇内碎片大，浪费10%~20%空间

#### FAT32
- 每个FAT表项占4字节（实际用28位）
- 簇大小4KB时，单个最大磁盘空间 = 4KB × 2^28 = **1TB**
- 可用较小簇，减少簇内碎片

**总结**：
- FAT12 → FAT16 → FAT32：表项位数增大，可管理磁盘容量增大
- 但FAT表存储开销也随之增大

### 8.2 混合索引（UNIX inode）

UNIX System V 的inode中设**13个地址项**：
- **直接地址**：i.addr(0) ~ i.addr(9)，共10个，直接指向数据块
- **一次间接**：i.addr(10)，指向一个索引块，该索引块存放数据块地址
- **二次间接**：i.addr(11)，指向一个索引块，该索引块指向多个索引块
- **三次间接**：i.addr(12)，三级索引

**最大文件大小计算**：
- 设盘块大小为4KB，每个地址占4字节
- 一个索引块可存 4KB/4B = 1024 个地址
- 直接：10 × 4KB = 40KB
- 一次间接：1024 × 4KB = 4MB
- 二次间接：1024 × 1024 × 4KB = 4GB
- 三次间接：1024^3 × 4KB = 4TB

### 8.3 文件物理结构比较

| 分配方式 | 优点 | 缺点 |
|----------|------|------|
| 连续分配 | 速度快，支持随机访问 | 产生碎片，不利于增长 |
| 链接分配 | 无碎片，利于增长 | 随机访问慢，指针占用空间 |
| 索引分配 | 支持随机访问，无碎片 | 索引块开销，访问需读索引 |

|------|------|

| 页号 P = INT[A / L] | A为逻辑地址，L为页大小 |
| 页内偏移 d = A MOD L | |
| 物理地址 = 物理块号 × 页大小 + 页内偏移 | 基本分页地址变换 |
| 缺页率 = 缺页次数 / 总访问次数 | |

## 复习建议

1. **概念题重点**：进程定义/状态/PCB、死锁四条件、4种I/O控制方式、文件逻辑结构
2. **计算题重点**：PV操作（水果盘）、调度算法（FCFS/SJF/高响应比）、地址变换、页面置换（FIFO/OPT/LRU）、缺页率
3. **公式背牢**：
   - 响应比 = (等待时间 + 服务时间) / 服务时间
   - 页号 = INT[逻辑地址 / 页大小]，偏移 = 逻辑地址 MOD 页大小
   - 缺页率 = 缺页次数 / 总访问次数
4. **FAT和混合索引**：注意大小计算、优缺点对比
